# 研究简报 Skill

## Skill 目标

围绕用户给定的主题，快速产出一份可作为后续学习、写作或讨论输入的研究简报。

这份简报不追求面面俱到，而是优先帮助用户建立基本地图：这个主题是什么、有哪些关键概念、常见分歧在哪里、下一步还该看什么。

## 何时使用

适用于以下场景：

- 用户说「帮我快速了解一下这个主题」
- 用户说「先做个研究简报」
- 用户说「给我一个这个话题的入门地图」
- 用户准备开始学习、写作、汇报或讨论一个新主题

不适用于：

- 只需要一句定义
- 需要最终成稿的长文写作
- 需要个性化训练计划的场景，那应交由学习计划类 Skill 处理

## 输入

优先从当前对话中获取：

- 主题名称
- 用户的目标：入门、复习、输出文章、准备面试、准备分享等
- 预期深度：速览、入门、深入
- 约束条件：时间、语言、使用场景、是否需要术语解释

可选输入：

- 用户已有基础
- 用户明确指定要覆盖或不要覆盖的子主题
- 用户已有资料、笔记、链接、附件

信息不足时，至少要明确主题本身；其他缺失项可先按入门默认处理。

## 输出

输出一份 Markdown 研究简报，建议包含：

1. 主题一句话定义
2. 为什么值得学
3. 5-8 个核心概念
4. 关键术语表
5. 常见误解或易混点
6. 延伸问题或后续可深挖方向

如果运行环境需要结构化输出，可返回：

```yaml
artifact:
  type: research_brief
  format: markdown
metadata:
  skill_name: research_brief_skill
  status: success_or_partial_or_paused
context:
  inputs_used: []
  tools_called: []
errors:
  warnings: []
```

## 核心流程

1. 确认主题和目标。
2. 判断用户是要速览、入门还是深入。
3. 按主题拆出核心概念，不直接堆资料。
4. 提炼术语、误区和关键问题。
5. 用适合后续 Skill 消化的方式输出简报。

## 质量约束

- 先搭地图，再补细节。
- 不用术语堆砌掩盖不确定性。
- 主题有多个流派或定义时，要明确指出，而不是混成一个版本。
- 不编造用户未提及的个人背景。
- 若信息不足，默认按「给零基础用户的入门简报」处理，并显式说明。

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 主题过大，例如「哲学」「AI」 | 先建议缩小范围，同时可先给总览版 |
| 用户目标冲突，例如既要 3 分钟看完又要全面深入 | 先满足速览，再列深入方向 |
| 主题缩写有歧义 | 先列出候选含义并请求确认 |
