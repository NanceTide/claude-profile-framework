# 法务合规 Skill

## Skill 目标

从产品立项视角识别法律、监管、数据安全与知识产权风险，输出 Charter 可用的「合规与风险」章节，确保立项材料在提交评审前覆盖常见合规检查项。

本 Skill 提供**立项级合规清单与风险提示**，不构成正式法律意见。

---

## 何时使用

- Charter、立项报告提交评审前的合规章节
- 用户提到「合规」「法务」「隐私」「知识产权」「监管」
- 多 Skill 流程末段，在业务与财务章节完成后串行执行（Profile 可标为可选）

不单独用于：
- 起草正式合同、律师函 → 需专业法务介入
- 仅咨询一条法条含义 → 简要说明即可

---

## 输入

**必需：**
- 产品/业务描述（功能、数据处理方式、目标市场）
- 前述章节的摘要：市场范围、竞品模式、财务投入规模（至少产品描述）

**可选：**
- 目标上线国家/地区列表
- 涉及的数据类型（个人信息、金融、医疗、未成年人等）
- 企业已有合规政策、DPA 模板、知识产权清单

**信息不足时：**
- 缺少目标市场 → 按「最严格合理假设」列风险，标「待确认适用法域」
- Profile 允许跳过（如内部草稿）→ 返回 skip

---

## 输出

Markdown 格式「合规与风险」章节，包含：

1. **适用法规与标准概览** — 按法域/行业列举可能适用的框架（如 GDPR、个保法、等保等，按实际范围）
2. **合规检查清单** — 表格：检查项 | 现状/计划 | 风险等级 | 建议动作
3. **知识产权** — 商标、专利、开源协议、第三方 SDK 等注意事项
4. **数据与隐私** — 收集范围、存储、跨境、用户同意机制
5. **立项建议** — 评审前必须完成的 3–5 项动作

文末说明：**本章节为风险提示，不构成法律意见；重大事项需法务正式评审。**

---

## 核心流程

### 第一步：界定范围

1. 从产品描述提取：行业、地域、数据类型、商业模式
2. 读取 Profile 中的强制合规项与禁忌措辞
3. 判断是否为 Profile 定义的「可跳过」场景

### 第二步：风险识别

1. 按监管、隐私、IP、劳动、广告促销等维度扫描
2. 结合竞争分析中的竞品合规实践（公开信息）
3. 对每项评估：高/中/低 + 缓解建议

### 第三步：输出清单

1. 生成检查清单与建议动作
2. 标注需外部法务或 DPO 确认的事项
3. 与财务章节的合规成本科目交叉引用（如有）

### 第四步：交接

1. 输出章节摘要供全文汇合
2. 列出「评审前必办」事项

---

## 质量约束

- **禁止假装法律意见**：统一免责声明
- **禁止编造法条或案例**：不确定则标「需法务确认」
- **与业务一致**：不引入与产品描述无关的合规项
- **Profile 措辞**：遵守交付规范中的禁忌词（如「亏损」→「投入期」由 Profile 规定）

---

## Memory 使用

可读：企业默认法域、已有认证（ISO27001 等）、标准合同流程、法务联系人（仅作「建议咨询 XX」提示）。

不可替代：正式法务审批结论。

---

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 涉及高风险行业（金融、医疗、未成年人） | 升高风险等级，建议正式法务评审 |
| Profile 跳过法务 | 输出 skip，主文档合规节留「内部草稿，未做法务审查」 |
| 多法域冲突 | 分法域列表，不合并为一句模糊结论 |
| 用户要求「保证合规」 | 说明 Skill 仅能提供检查清单，无法保证 |
