# 学习计划 Skill

## Skill 目标

把一个主题的研究简报转化为可执行的学习路径，让用户知道先学什么、后学什么、每一阶段要达到什么程度。

本 Skill 重点是编排学习顺序和节奏，而不是重复解释主题内容。

## 何时使用

适用于以下场景：

- 用户说「帮我制定学习计划」
- 用户已经有一份主题简报，希望继续拆成行动步骤
- 用户要在固定时间内学会某个主题

不适用于：

- 只有主题名，没有任何学习目标和时间约束，且用户也不接受默认值
- 用户只想要一个知识概览，不想进入计划阶段

## 输入

必需输入：

- 主题研究简报或等价摘要

强烈推荐输入：

- 用户当前基础：零基础、初级、中级
- 学习目标：入门、项目实战、考试、面试、写作、授课
- 可投入时间：每天/每周多久，总周期多长

可选输入：

- 偏好形式：看文、做题、上手实践、对话讲解
- 明确不喜欢的方式
- 截止日期

若缺少这些信息，可按默认值处理：

- 默认基础：零基础
- 默认节奏：每周 4 次，每次 45 分钟
- 默认周期：2 周

## 输出

输出一份 Markdown 学习计划，建议包含：

1. 学习目标
2. 先修假设
3. 阶段划分
4. 每阶段任务
5. 每阶段完成标准
6. 复习与检查点
7. 时间安排建议

## 核心流程

1. 读取研究简报，识别核心概念之间的依赖关系。
2. 根据用户基础和目标决定教学顺序。
3. 把学习任务切成阶段，避免一次吞太多。
4. 为每阶段写出完成标准，而不仅是任务清单。
5. 预留复习和回顾节点。

## 质量约束

- 计划必须可执行，不能只是抽象口号。
- 顺序必须说明为什么这样排。
- 每阶段任务数量适中，适合真实生活节奏。
- 对零基础用户不要默认已掌握前置知识。
- 不伪造外部课程或书目；如未指定资料，就写资料类型而不是硬编名称。

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 用户时间极少 | 压缩为最小可行学习路径 |
| 用户目标很高但周期很短 | 明确指出取舍，优先保核心能力 |
| 研究简报本身范围过宽 | 先提醒收窄，再给保守版计划 |
