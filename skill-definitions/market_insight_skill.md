# 市场洞察 Skill

## Skill 目标

针对某一产品或业务方向，系统分析目标市场规模、增长趋势、用户画像、需求痛点与机会窗口，输出可直接嵌入立项文档的「市场洞察」章节。

本 Skill 提供标准方法论与输出结构；具体信源偏好、覆盖区域、深度要求由 Profile 覆盖。

---

## 何时使用

- 撰写产品 Charter、立项报告、商业计划书的市场章节
- 用户要求分析「市场机会」「目标用户」「行业趋势」
- 作为多 Skill 串联流程中的第一阶段输入

不单独用于：
- 仅查单个数据点（如「2025 年 SaaS 市场规模是多少」）→ 简单检索即可
- 纯竞品对比 → 使用竞争分析 Skill

---

## 输入

**必需：**
- 产品/业务名称或方向描述
- 目标市场范围（地域、行业、用户群，至少一项）

**可选：**
- 已有调研材料、行业报告、用户访谈摘要
- Profile 指定的信源白名单/黑名单
- 时间范围（如「近 3 年趋势」「2026–2028 预测」）

**信息不足时：**
- 缺少目标市场 → 追问或基于产品描述做合理假设并标注「待确认」
- 缺少产品方向 → 暂停，请用户说明

---

## 输出

一份 Markdown 格式的「市场洞察」章节，包含以下必选小节：

1. **市场概况** — 市场规模、增速、主要驱动因素（标注数据来源或「待补充」）
2. **目标用户画像** — 1–2 个核心用户群：角色、痛点、使用场景
3. **需求与机会** — 未被满足的需求、进入窗口、关键假设
4. **风险与不确定性** — 市场侧主要风险（政策、替代方案、周期等）

每节末尾可附「关键假设」列表，供后续财务预测 Skill 引用。

---

## 核心流程

### 第一步：明确分析边界

1. 确认产品/业务方向与目标市场
2. 确认分析深度（概览 vs 深度）与时间范围
3. 读取 Profile 中的信源偏好与 Memory 中的项目/部门信息

### 第二步：信息收集

1. 优先使用用户提供的材料与 Profile 指定信源
2. 必要时进行公开信息检索（行业报告、新闻、统计局数据等）
3. 记录每条关键数据的来源；无法核实则标「待补充」，不编造

### 第三步：结构化分析

1. 提炼市场规模与趋势（尽量量化：TAM/SAM/SOM 或等价表述）
2. 构建用户画像与痛点矩阵
3. 归纳机会点与进入时机
4. 列出市场侧风险与关键假设

### 第四步：输出与交接

1. 按输出模板生成章节
2. 在章节末尾输出「本章摘要」（3–5 条 bullet），供并行 Skill 汇合时使用
3. 标注需竞争分析或财务章节引用的假设编号

---

## 质量约束

- **禁止编造数据**：无来源的数字用「约」「待补充」或范围表述
- **区分事实与推断**：趋势判断需说明依据
- **用户导向**：避免空泛行业描述，紧扣目标用户与产品方向
- **可引用性**：关键假设编号化（如 M1、M2），便于下游 Skill 引用

---

## Memory 使用

可读：项目名称、所属事业部、目标区域偏好、历史 Charter 中的市场假设。

不可读：与本次分析无关的个人或财务敏感信息。

冲突时以用户当前输入与 Profile 为准。

---

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 公开数据 scarce | 用定性分析 + 「待补充」占位，列出建议补充的数据项 |
| Profile 禁止某信源 | 不使用该信源，并在输出中说明 |
| 市场定义模糊 | 给出 2 个可选市场边界方案，请用户择一或标注默认假设 |
| 与竞争分析并行 | 只输出市场侧内容，不抢写竞品对比（留给竞争分析 Skill） |
