# 会议纪要整理

## Skill 目标

将会议录音转写文本、手写笔记、聊天记录等原始素材，整理为结构规范、重点突出、可归档可发送的正式会议纪要文档。忠实反映会议讨论内容，不编造、不遗漏。

## 何时使用

- 会议结束后需要整理纪要归档
- 需要将会议成果同步给未参会人员
- 项目关键决策需要书面记录备查
- 周会、评审会、复盘会、头脑风暴等常规会议

不要用于：
- 非正式闲聊、无决策无任务的随意交流
- 仅有"帮忙记一下"但没有提供任何素材的情况

## 输入

**必须提供：**
- 会议原始素材（录音转写文本 / 手写笔记 / 聊天记录 / 口述要点，至少一种）
- 会议主题

**可选提供：**
- 会议日期、参会人员、会议类型
- 公司或团队的纪要模板
- 会议 PPT、文档等参考资料

**信息不足时：**
- 核心信息缺失（素材和主题都没有）→ 先追问再启动
- 行动项缺负责人或截止时间 → 追问确认，不编造
- 参会人不明确 → 追问确认

## 输出

一份结构完整的会议纪要 Markdown 文档，包含四个必选章节：

1. **会议概要** — 1-2 句话概括核心议题和结论
2. **讨论要点** — 按议题分组，区分"已达成共识"和"存在分歧"
3. **行动项** — 表格形式，每条含事项、负责人、截止时间
4. **会议决议** — 只列有明确结论的事项，未定论的归入"待讨论"

## 核心流程

### 第一步：信息收集

1. 确认会议主题、日期、参会人
2. 获取会议原始素材
3. 确认会议类型（周会/评审会/复盘会/头脑风暴），以便调整纪要风格
4. 确认是否有特殊格式要求

### 第二步：内容提炼

1. **提炼概要** — 1-2 句话，标注是否达成共识
2. **整理讨论要点** — 按议题分组，记录各方观点和反对意见，区分"共识"和"分歧"
3. **梳理行动项** — 每条含事项、负责人、截止时间；缺失则追问，无法确认标注"待确认"
4. **记录决议** — 只写有明确结论的事项
5. **补充信息**（可选）— 关键数据、参考资料链接、下次会议时间

### 第三步：格式检查

1. 四个必选章节是否齐全
2. 行动项是否有负责人和截止时间
3. 是否忠实还原原始素材
4. 是否有编造的内容

### 第四步：输出确认

1. 展示完整纪要
2. 询问是否需要调整
3. 确认后保存文件

## 质量约束（强制）

- **禁止编造**：所有内容必须有素材依据，不得编造讨论、结论或决议
- **行动项必有归属**：每条行动项必须有负责人和截止时间，缺失则追问
- **区分共识与分歧**：不得将个人观点等同于集体决策，未定论事项须显性标注
- **禁止主观判断**：使用第三人称描述观点（"张三提出""李四建议"），不加整理者评价
- **行动项具体可执行**：禁用"跟进""关注"等模糊动词

## 会议类型适配

- **周会/站会**：简洁，重点在进度和阻塞项
- **评审会**：详细，重点在评审意见和修改要求
- **复盘会**：重点在问题根因和改进措施
- **头脑风暴**：忠实记录所有观点，不做筛选排序
- **1on1**：重点在讨论共识和个人反馈

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 原始素材太少，无法构成完整纪要 | 逐项追问，至少补齐主题、结论和行动项再开始 |
| 素材杂乱无章 | 先按议题分类，结果给用户确认后再撰写 |
| 大量闲聊或跑题内容 | 只提取与议题相关的讨论，忽略闲聊 |
| 行动项缺负责人或截止时间 | 追问确认，不编造 |
| 同一议题有多个方案未定论 | 如实记录各方观点，归入"待讨论事项" |
| 素材涉及敏感信息 | 照常写入但提示用户在发送前确认 |
