# 财务预测 Skill

## Skill 目标

基于市场与竞争分析结论，构建产品立项阶段的财务视图：成本结构、收入假设、关键指标（ROI、回收期、盈亏平衡点等），输出 Charter 可用的「财务预测」章节。

本 Skill 侧重**立项级粗算与假设透明**，非精确会计或审计级模型。

---

## 何时使用

- Charter、立项报告、商业计划中的财务/投入产出章节
- 用户要求「算一下 ROI」「投入多少、多久回本」「做个财务模型」
- 多 Skill 流程中，在市场洞察 + 竞争分析完成后串行执行

不单独用于：
- 上市公司财报解读 → 简单分析即可
- 已定型产品的详细预算编制 → 需更细粒度模板

---

## 输入

**必需：**
- 产品/项目方向与预期交付范围（MVP 或 Phase 1）
- 市场洞察与竞争分析章节的摘要或全文（至少其一）

**可选：**
- 用户提供的预算上限、人力编制、定价意向
- ERP/财务系统数据（Profile 指定；无权限则标「待补充」）
- 时间 horizon（默认 3 年或 Profile 指定）

**信息不足时：**
- 缺少上游章节 → 可基于用户口述假设启动，但须显式列出假设且标「待上游确认」
- 关键数字缺失 → 用区间或「约 xx 万」，禁止伪精确

---

## 输出

Markdown 格式「财务预测」章节，包含：

1. **建模假设** — 表格：假设项 | 取值 | 依据 | 引用（M1/C1 等）
2. **成本估算** — 研发、运营、市场、人力等（分项 + 合计，允许粗算）
3. **收入预测** — 定价逻辑、用户/订单量假设、分年收入（如适用）
4. **关键指标** — ROI、回收期、盈亏平衡、敏感性说明（1–2 个关键变量）
5. **财务风险与待确认项** — 需用户或财务确认的条目

默认 Skill 节奏：**生成完整章节后建议暂停待用户确认数字**（Profile 可覆盖）。

---

## 核心流程

### 第一步：对齐假设

1. 读取市场洞察中的规模/用户假设、竞争分析中的定价参考
2. 向用户或 Memory 确认预算约束、币种、时间范围
3. 建立假设登记表（编号 F1、F2…）

### 第二步：成本建模

1. 按立项粒度拆分：一次性 vs  recurring
2. 人力按角色×人月或 Profile 指定口径
3. 缺数据时用区间，并说明敏感度

### 第三步：收入与指标

1. 定价策略对齐竞争分析结论
2. 分年（或分阶段）收入与成本
3. 计算 ROI、回收期等；公式与假设一并写出

### 第四步：输出与确认

1. 生成章节 + 假设总表
2. 高亮 3 个最敏感假设
3. 若 Profile 要求暂停 → 输出后等待用户确认再继续后续 Skill

---

## 质量约束

- **假设可追溯**：每个重要数字能追溯到 M/C/F 假设或用户输入
- **粗算诚实**：立项阶段允许 ±30% 量级，禁止虚假小数精度
- **不编造权限外数据**：ERP 无数据则标黄「待补充」
- **与法务分工**：合规成本可预留科目，详细法条由法务合规 Skill 处理

---

## Memory 使用

可读：部门预算习惯、默认币种、历史项目成本量级、审批人关注的指标（如必须展示 ROI）。

不可编造：未授权的实际财务数据、已审批预算数字。

---

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 上游章节缺失 | 基于用户口述列假设，全文标「待市场/竞品章节对齐」 |
| 用户否定某假设 | 更新假设表并重算指标 |
| 竞争分析被跳过 | 定价假设仅依赖市场洞察与用户输入 |
| 回收期为负或显然不合理 | 指出矛盾，建议调整假设而非强行美化 |
