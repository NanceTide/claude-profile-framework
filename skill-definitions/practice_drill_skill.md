# 练习与自测 Skill

## Skill 目标

把学习计划变成可检验的练习与自测材料，帮助用户确认自己不是“看懂了”，而是真的能解释、判断和应用。

## 何时使用

适用于以下场景：

- 用户说「给我出点练习」
- 用户已经有研究简报和学习计划，希望增加自测
- 用户要准备面试、考试、分享或输出文章，需要检验掌握程度

不适用于：

- 用户还没理解基本概念
- 用户只要阅读清单，不需要练习

## 输入

必需输入：

- 主题研究简报
- 学习计划或阶段安排

可选输入：

- 用户偏好的练习形式：问答、判断、案例、写作、口头讲解
- 练习数量
- 难度：基础、进阶、混合

若无额外说明，默认输出混合难度的 3 类练习：

- 概念解释
- 应用判断
- 输出任务

## 输出

输出一份 Markdown 练习包，建议包含：

1. 5-8 题自测题
2. 2-3 个应用题或案例题
3. 1 个输出型任务
4. 1 份自查清单
5. 简短答案要点或评分标准

## 核心流程

1. 读取研究简报中的核心概念和误区。
2. 对照学习计划中的阶段目标出题。
3. 题目从记忆、理解、应用三个层次分布。
4. 补一个输出型任务，检查迁移能力。
5. 给出简短答案要点，方便用户自查。

## 质量约束

- 题目要覆盖关键概念，而不是只考边角知识。
- 输出型任务要能逼近真实使用场景。
- 不把答案写成长篇讲义。
- 不生成只有定义复述的低质量题海。

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 用户还没学完基础 | 只出基础题，并提醒先完成前两步 |
| 用户没有说明练习形式 | 提供混合题型 |
| 学习计划过粗 | 先按主题核心能力出通用练习 |
