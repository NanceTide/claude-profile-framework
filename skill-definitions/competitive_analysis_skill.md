# 竞争分析 Skill

## Skill 目标

识别直接/间接竞争对手，对比产品能力、定价、渠道与差异化，输出立项文档可用的「竞争分析」章节，并给出可执行的差异化建议。

---

## 何时使用

- 撰写 Charter、立项报告、产品规划中的竞品章节
- 用户提到「竞品」「竞争对手」「对标分析」
- 多 Skill 流程中与市场洞察并行执行

不单独用于：
- 仅查某公司股价或新闻 → 简单检索
- 纯市场规模分析 → 使用市场洞察 Skill

---

## 输入

**必需：**
- 产品/业务方向（可与市场洞察章节一致）
- 至少 1 个已知竞品名称，或明确的竞品发现范围

**可选：**
- 用户指定的必分析竞品列表
- 市场洞察章节的摘要或假设（并行汇合后作为输入）
- 竞品官网、发布会、评测文章等材料

**信息不足时：**
- 用户未提供任何竞品且无法从上下文推断 → 先做 3–5 家候选竞品发现，标注「待用户确认」
- Profile 规定「无竞品则跳过」→ 返回 `skipped` 状态与简要说明

---

## 输出

Markdown 格式「竞争分析」章节，包含：

1. **竞争格局概览** — 主要玩家、市场分层（领导者/挑战者/利基）
2. **竞品对比表** — 至少 2–3 家：定位、核心功能、定价模式、优势、劣势
3. **差异化机会** — 基于对比得出的 2–4 条可落地差异点
4. **竞争风险** — 跟进、价格战、生态封锁等

文末附「本章摘要」供财务预测与全文汇合使用。

---

## 核心流程

### 第一步：确定竞品范围

1. 从用户输入、Memory、市场方向推断竞品候选
2. 按 Profile 要求确定必分析名单（如「必须覆盖 A 公司最近发布会」）
3. 若无任何可分析竞品且 Profile 允许跳过 → 终止并说明

### 第二步：信息收集

1. 公开信息：官网、文档、新闻、招聘、用户评价
2. 不编造未公开的内部数据、未发布功能或精确财务
3. 信息时效性标注（如「截至 2026-Q1」）

### 第三步：对比与结论

1. 填写竞品对比维度（功能、价格、渠道、客户群、技术栈等，按 Profile 裁剪）
2. 归纳差异化机会与竞争壁垒
3. 列出需持续跟踪的竞争动态

### 第四步：输出与交接

1. 生成章节与摘要
2. 将「定价/商业模式」相关结论标记供财务预测 Skill 引用

---

## 质量约束

- **禁止编造竞品能力**：未证实功能标「待核实」
- **客观对比**：避免贬损性措辞，用事实与维度对比
- **可行动**：差异化建议需与自家产品方向可衔接
- **与市场洞察分工**：不写市场规模章节，可交叉引用市场假设编号

---

## Memory 使用

可读：历史竞品名单、部门关注的对标公司、项目代号。

不可编造：竞品未公开的销量、收入、内部 roadmap。

---

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 竞品信息极少 | 基于公开页面做有限对比，缺失维度标「信息不足」 |
| 用户指定竞品不存在或拼写错误 | 列出最接近匹配，请用户确认 |
| 并行于市场洞察 | 可独立启动；汇合前避免重复写市场概况 |
| Profile 要求跳过 | 输出 skip 说明，不生成完整章节 |
