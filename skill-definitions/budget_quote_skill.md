# 预算报价 Skill

## Skill 目标

围绕项目范围、交付内容和资源投入，生成可用于客户提案、内部评审或商务沟通的预算与报价说明。

本 Skill 负责把预算相关信息整理成结构化说明、报价版本或费用拆分表，不替用户编造价格、成本依据、付款条件或折扣策略。

---

## 何时使用

- 用户要求补预算、报价、费用拆分、版本报价
- 解决方案正文需要附带预算或商务说明
- 内部评审需要估算成本、报价区间或资源投入

不单独用于：
- 只写方案正文 -> 使用解决方案提案 Skill
- 只做财务长期测算 -> 使用财务预测 Skill
- 只做合同条款审阅 -> 使用法务合规 Skill

---

## 输入

**必需：**
- 项目范围或交付内容
- 至少一种预算依据：单价、资源投入、工期、已有报价口径、目标总价

**可选：**
- 解决方案正文摘要
- 资源角色与投入工时
- 采购或外包成本
- 版本方案（基础版、标准版、高配版）
- 付款节奏、折扣空间、税费说明

**信息不足时：**
- 缺少价格依据 -> 不给确定报价，只给结构和待确认项
- 缺少项目范围 -> 先要求确认范围，否则报价失真

---

## 输出

根据用户需要输出以下一种或多种结果：

1. **预算摘要** — 说明报价逻辑、范围和关键假设
2. **费用拆分表** — 按模块、阶段、角色或成本项展开
3. **版本报价对比** — 基础版 / 标准版 / 高配版等
4. **报价说明文本** — 供放入方案正文或对外说明
5. **待确认清单** — 尚未确定、不能硬报的项目

默认推荐包含以下字段：

| 项目项 | 说明 | 计价依据 | 金额/区间 | 备注 |
|--------|------|----------|----------|------|

---

## 核心流程

### 第一步：确认报价边界

1. 明确本次是内部估算、客户正式报价，还是仅做预算占位
2. 明确项目范围、交付模块、时间周期和排除项
3. 区分已确认价格依据与待确认部分

### 第二步：建立报价结构

1. 按模块、阶段、角色或版本拆分预算
2. 对每项写明计价依据
3. 识别哪些内容应纳入，哪些应排除或单独报价

### 第三步：生成报价内容

1. 给出预算摘要和费用拆分
2. 若存在多版本方案，则生成对比表
3. 对不确定部分使用区间、占位或待确认说明

### 第四步：校验稳妥性

1. 检查是否与方案正文范围一致
2. 检查是否出现无依据的总价或折扣
3. 对外版本避免过满承诺，保留假设与边界

### 第五步：输出与交接

1. 输出预算页、报价说明或费用表
2. 把关键假设和限制条件单列
3. 若要对外发送或进入正式审批，则交给后续审校环节

---

## 质量约束

- 不编造价格、折扣、税率、付款条件
- 不脱离项目范围单独报总价
- 每个金额或区间尽量有明确依据
- 不确定项应显式标注“待确认”“暂估”“区间”
- 对外版本要写清适用范围和边界条件

---

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 用户只说“给个报价” | 先追问项目范围、交付内容和用途 |
| 只能给粗略预算 | 输出区间估算，并列关键假设 |
| 多版本差异不清 | 先列版本差异框架，再补金额 |
| 方案正文未定 | 只给预算结构和待确认项，不输出正式报价 |
| 用户要求直接对外发报价 | 先检查依据是否齐全，不足则明确风险 |
