# 通用主题学习路径 Profile

## 适用场景

我说「帮我快速学会一个主题」「给我一套学习路径」「先研究再帮我安排学习」「我要系统学这个东西」时启用。

主题可以是技术、学科、语言、写作方法、研究方向或任何需要从陌生到可用的内容，不限定企业内部流程。

如果我只是问一句定义、要一个摘要，停在研究简报阶段即可，不强行进入后续学习计划和练习。

如果我只是说「给我出题」，优先匹配练习相关场景；若同时像「系统学习」又像「只做练习」，先列出候选 Profile 让我选。

## 技能依赖

**必选：** 研究简报 Skill、学习计划 Skill、练习与自测 Skill。

三个 Skill 默认串行使用。前一个 Skill 的输出是后一个 Skill 的输入，不并行执行。

## 目标标准

目标不是只给我一堆资料，而是把一个主题拆成三层成果：

- 我先看懂这个主题的大图景和关键术语
- 我知道接下来按什么顺序学习、每阶段学到什么程度
- 我能通过练习判断自己是否真的掌握

必须覆盖：

- 主题定义与核心概念
- 阶段化学习路径
- 阶段完成标准
- 自测题、应用题或输出任务

**取舍原则：** 清晰路径优先于资料堆砌；可执行优先于“看起来很全面”。

## 流程编排

1. **串行：研究简报 Skill。**
   - 输入：主题名、用户目标、已有基础、时间约束、已有资料。
   - 输出：主题地图、核心概念、关键术语、误区、延伸问题。
   - 如果主题过大或缩写有歧义，先收窄范围或确认含义。

2. **检查点：范围确认。**
   - 展示研究简报摘要。
   - 如果我说主题范围太大、太浅或方向不对，先改研究简报，再进入下一步。

3. **串行：学习计划 Skill。**
   - 输入：研究简报 + 我的基础 + 学习目标 + 时间节奏。
   - 输出：阶段化学习计划、每阶段任务、完成标准、复习节点。
   - 若我没给时间安排，按默认节奏生成保守版计划。

4. **检查点：计划确认。**
   - 展示学习计划摘要和总周期。
   - 如果我说时间太长、太短或过难，先调整计划，再进入下一步。

5. **串行：练习与自测 Skill。**
   - 输入：研究简报 + 最终学习计划。
   - 输出：自测题、应用题、输出任务、自查清单。
   - 若我明确说不要练习，可跳过这一步。

6. **汇合：最终交付。**
   - 按顺序输出研究简报、学习计划、练习包。
   - 最后列出我下一次继续学习时最值得带上的上下文信息。

## 协作规范

**检查点：**

- 研究简报完成后必须暂停一次，确认方向。
- 学习计划完成后再暂停一次，确认节奏。
- 练习包可以在确认计划后一次性生成。

**汇报：**

- 每一步先给摘要，再给完整结果。
- 如果某一步超过十分钟没有成型结果，要告诉我卡在主题范围、时间约束还是资料不足。

**异常：**

- 主题太大：先帮我缩成一个可学的切片。
- 我的目标和时间冲突：直说冲突，不硬凑“完美计划”。
- 资料缺失：先给通用版本，再标出最值得补的资料类型。
- 主题存在多种流派：并列写出，不强行合并。

## 知识预设

自动从 Memory 或当前对话取用：

- 我的背景水平：零基础、初级、中级
- 我的常用学习节奏，例如每天 30 分钟、周末集中学
- 我的偏好形式，例如先看概念、再做练习，或者先上手实践
- 我最近正在学的长期主题，避免每次重复介绍上下文

值得写入 Profile Memory 的事实：

- 某个长期学习主题
- 常见时间预算
- 偏好题型
- 偏好的输出形式，例如 Markdown、清单、表格

不要写入 Profile Memory：

- 某次临时学习的具体截止时间
- 某次临时考试日期
- 一次性的练习答案

## 交付规范

- 输出格式：Markdown
- 最终交付固定分三段：研究简报、学习计划、练习与自测
- 研究简报更像“主题地图”，不要写成长篇教科书
- 学习计划按阶段编号，每阶段写目标、任务、完成标准
- 练习部分至少包含概念题、应用题、输出题三类
- 语气清晰、鼓励、不过度居高临下
- 不强行推荐具体课程、书籍或平台；如果没有明确来源，就写资料类型
