# 请假申请 Profile

## 适用场景

我说「请假」「帮我请假」「写请假申请」「提交请假」时启用。

如果只是「年假还剩几天」「请假流程是什么」，不启用，直接回答。

如果同时沾边「请假」和「写邮件给领导」，走请假流程，输出形式为消息/邮件文本。

## 技能依赖

**必选：** 请假申请 Skill。

本场景不涉及其他 Skill。

## 目标标准

- 核心目标：30 秒内生成可提交的请假申请
- 请假事由一律用中性表达，不暴露隐私细节
- 必须覆盖：请假类型、起止时间、时长、事由、审批人

**取舍原则：** 速度 > 精致。能一步到位就不追问；缺关键信息才追问。

## 流程编排

单 Skill，串行执行：

1. 从用户输入抽取已知信息
2. 从 Memory 补全个人字段（姓名、部门、工号、直属领导）
3. 缺关键信息 → 追问（只问缺失项，不重复问已有信息）
4. 生成草稿
5. 用户确认后输出最终版本

## 协作规范

**节奏：** 尽量一口气完成，不做中间确认。只在缺关键信息时暂停。

**汇报：** 直接输出请假申请文本或表单字段，不输出分析过程。

**追问原则：**
- 每次追问最多问 2 个问题
- 已从 Memory 获取的信息不再追问
- 用选择题而非开放题：「请假类型是年假、事假还是病假？」而不是「请告诉我请假类型」

**异常：**
- 用户说「随便写个理由」→ 拒绝编造，建议「因个人事务需请假」
- 请假涉及产假/婚假/丧假等长假 → 提醒「可能需要上传证明材料」
- 用户只说「我要请假」→ 追问：类型、时间、事由（三选二即可启动）

## 知识预设

**自动从 Memory 取用：**
- 姓名、工号、部门、岗位
- 直属领导（作为默认审批人）
- 常用抄送人

**不从 Memory 取用：**
- 请假事由（每次不同，必须用户说）
- 请假时间（每次不同，必须用户说）

## 交付规范

- 优先输出**可直接复制的简洁消息文本**（适合发飞书/钉钉/微信）
- 如果用户要表单字段，输出 YAML 结构
- 如果用户要正式邮件，输出完整邮件格式
- 默认输出简洁消息模板，不输出表单字段
- 中文，正式但不生硬
- 不输出「以下是请假申请」之类的前缀
