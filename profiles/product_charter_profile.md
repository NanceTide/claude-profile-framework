# 产品立项 Charter Profile

## 适用场景

我说「写 Charter」「立项报告」「产品规划」「帮我写个立项文档」「PRD 立项」时启用。

若我说「查一下市场规模」「随便看看竞品」，不走完整 Charter 流程，只调用对应单项能力或简单检索。

若同时像「写 Charter」又像「写周报」，先列出候选项让我选。

内部草稿、仅自己看时可说「内部版 Charter」——此时法务合规 Skill 可跳过（见技能依赖）。

## 技能依赖

**必选：** 市场洞察、竞争分析、财务预测。

**可选：** 法务合规。内部草稿、或用户明确说「先不要法务」时可跳过。

## 目标标准

目标是产出**可支撑立项评审会决策**的产品 Charter，读者是产品负责人与跨部门评审委员。

必须回答：

- 做什么产品、给谁用、解决什么痛点（市场洞察）
- 竞品是谁、我们差异在哪（竞争分析）
- 大概投多少、多久见效、关键假设是什么（财务预测）
- 有哪些合规红线要前置处理（法务合规，若未跳过）

必须覆盖 Profile 或对话中明确的项目范围（如「东南亚市场」「云原生」「Q3 里程碑」——以用户当次说明为准）。

**取舍原则：** 深度优先于速度；成本允许粗算，但**关键假设必须写明**；数据缺则用「待补充」占位，不编造。

## 流程编排

1. **并行：** 市场洞察 + 竞争分析（两路同时跑，均完成后进入下一步）。
   - 竞争分析输入：产品方向 + 用户提供的竞品名单；可引用对话上下文，不强制等待市场洞察全文。
   - **分支：** 若用户明确「没有竞品」或 Profile 判定无可用竞品信息 → **跳过**竞争分析，其余顺序不变。

2. **串行：** 财务预测（输入：市场洞察 + 竞争分析两章产出或摘要；若竞争分析已跳过，则仅输入市场洞察 + 用户补充）。

3. **串行：** 法务合规（输入：产品描述 + 前三章摘要；**可选**，见技能依赖）。

4. **汇合：** 将各章整合为完整 Charter Markdown 文档，结构如下：
   - 文档摘要（Executive Summary，5–8 条）
   - 一、市场洞察
   - 二、竞争分析（若已跳过则写「本章略：无明确竞品」）
   - 三、财务预测
   - 四、合规与风险（若已跳过则标注「内部草稿，未做法务审查」）
   - 五、里程碑与下一步（Agent 整合，不另调 Skill）

## 协作规范

**检查点：**

- 步骤 1（市场 + 竞品）可一口气跑完，中间不暂停。
- 步骤 2（财务预测）**完成后必须暂停**，等我确认数字和假设后再继续法务与全文汇合。
- 全文拼接前先发**目录级大纲**让我扫一眼结构；我无异议再输出完整版。

**汇报：**

- 并行两章完成后，各发**章节摘要**（每章 3–5 条），不发全文。
- 财务章完成后发**全文**（含假设表）。
- 最终 Charter 完成后发摘要：共几章、几处「待补充」、是否跳过竞品/法务。

**异常：**

- 查不到数据 → 「待补充」占位，全文跑完再汇总标红清单。
- 财务假设与上游章节矛盾 → 指出冲突，请我选择以哪边为准。
- 法务章若委托「法务助理」且超时（Profile 默认 30 分钟无响应）→ 跳过法务章继续拼稿，并标注未审查。
- 听不懂需求 → 直接说，不硬猜。

## 知识预设

**信源：**

- 市场洞察优先查公开行业报告、统计局、主流科技媒体；**不要用 Gartner**（除非我另行指定）。
- 竞争分析若我提到具体公司，必须覆盖其**最近一场公开发布会或年度产品动态**（公开信息范围内）。
- 财务数据若接 ERP 无权限 → 标黄「待补充」，不编造。

**Memory 自动取用（不要重复问我）：**

- 我的姓名、部门、直属领导（摘要与文档署名）
- 当前对话中的**项目名、目标区域、技术栈关键词**
- 若 Memory 有「默认 Charter 模板路径或结构偏好」，遵循之

**附件：**

- 若对话中已上传 Q1/Q2 复盘、旧版 Charter，作为市场与财务的参考，但不直接复制过时数字。

## 交付规范

- 体裁：**内部立项汇报**，不要写成学术论文。
- 格式：完整 Markdown；先 Executive Summary，再分章；每章建议不超过两页等价篇幅（约 800–1000 字/章）。
- 数字：用「约 xx 万」「xx% 量级」，立项粗算不精确到小数点后两位。
- 能画结构图、里程碑时间线则用 Mermaid 或 ASCII 简图。
- **禁忌措辞：** 不出现「亏损」，改用「投入期」；避免「一定成功」「保证合规」等绝对化表述。
- 财务章与合规章保留假设编号（M/C/F/L），便于评审追问。
